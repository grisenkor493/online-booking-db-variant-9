Функциональные тест-кейсы (Бизнес-операции)

    TC-01: Создание записи на свободный слот

        Действие: Перейти в «Онлайн-запись» -> «Создать новую запись». Выбрать клиента, курс, дату и свободное время. Нажать «Подтвердить».

        Ожидаемый результат: Запись успешно сохраняется, появляется зеленое уведомление, генерируется код бронирования, запись отображается в журнале со статусом «Ожидает».

    TC-02: Динамическая загрузка временных слотов (AJAX)

        Действие: В форме создания записи выбрать дату.

        Ожидаемый результат: Появляется индикатор загрузки (спиннер) и подгружаются кнопки/радиокнопки с доступным временем. Страница при этом не перезагружается полностью.

    TC-03: Защита от двойного бронирования (Конкурентная защита)

        Действие: Попробовать вручную или через вторую вкладку симулировать отправку формы на то же самое время, которое только что было успешно забронировано.

        Ожидаемый результат: Система отклоняет операцию с ошибкой: «К сожалению, выбранное время только что занято другим клиентом...».

    TC-04: Серверная валидация даты (Запрет записи в прошлое)

        Действие: В поле даты выбрать вчерашний или любой прошедший день.

        Ожидаемый результат: Форма не отправляется, выводится ошибка валидации.

    TC-05: Смена статуса записи в журнале

        Действие: В таблице записей нажать кнопку «Подтвердить» или «Завершить» на активной строке.

        Ожидаемый результат: Статус записи меняется, цвет индикатора обновляется (например, зеленый для подтвержденных).

    TC-06: Отмена бронирования с освобождением слота

        Действие: Нажать кнопку «Отменить» на записи. Подтвердить действие во всплывающем окне браузера.

        Ожидаемый результат: Статус меняется на «Отменена» (красный индикатор). При повторной попытке создать запись на этот день отмененное время снова должно стать доступным для выбора.

    TC-07: Перенос времени записи (Reschedule)

        Действие: В таблице записей нажать кнопку «Перенести». Выбрать новую дату и время, подтвердить.

        Ожидаемый результат: Старое время освобождается, запись обновляется на новые параметры, в базе данных фиксируется лог изменений.

    TC-08: Фильтрация журнала записей

        Действие: Выбрать конкретную дату или определенный статус в блоке фильтров над таблицей и нажать «Применить».

        Ожидаемый результат: В таблице остаются только те записи, которые соответствуют выбранным условиям.

    TC-09: Просмотр аналитических отчетов

        Действие: Перейти во вкладку «Отчёты».

        Ожидаемый результат: Корректно выводятся две таблицы: распределение количества броней по дням и рейтинг популярности языковых курсов.

    TC-10: Экспорт отчета в формат CSV

        Действие: На странице отчетов нажать на кнопку «Скачать CSV-отчет».

        Ожидаемый результат: Браузер скачивает файл report.csv, в котором содержатся все данные из таблиц статистики.





        # Тест-план для проверки модуля онлайн-записи и отчетов

## 1. Введение
Данный тест-план разработан для проведения функционального тестирования бизнес-операций системы онлайн-записи: создание бронирований, управление статусами, перенос времени и генерация аналитических отчетов.

## 2. Перечень тестовых сценариев

### ТС-01: Создание записи на свободный слот
* **Предусловия:** В базе данных есть активные клиенты и курсы. Выбранная дата не находится в прошлом.
* **Шаги:**
  1. Перейти в «Онлайн-запись» -> «Создать новую запись».
  2. Выбрать клиента и курс из списков.
  3. Указать корректную дату.
  4. Выбрать один из появившихся слотов времени.
  5. Нажать кнопку «Подтвердить запись».
* **Ожидаемый результат:** Запись сохранена, отображается флеш-уведомление с кодом бронирования, запись появилась в журнале со статусом «Ожидает».

### ТС-02: Динамическая загрузка слотов через AJAX
* **Предусловия:** Страница создания записи открыта.
* **Шаги:**
  1. Кликнуть на поле выбора даты и выбрать день.
* **Ожидаемый результат:** Под полем даты появляется индикатор загрузки, затем отображаются доступные радиокнопки со временем. Полной перезагрузки страницы не происходит.

### ТС-03: Защита от конкурентных (двойных) бронирований
* **Предусловия:** Слот на 09:00 выбранного дня свободен. Открыто два окна браузера на этой странице.
* **Шаги:**
  1. В первом окне заполнить форму на 09:00 и нажать «Подтвердить».
  2. Во втором окне сразу же отправить форму на это же время (09:00).
* **Ожидаемый результат:** Первая запись создается успешно. Вторая отклоняется сервером с ошибкой «К сожалению, выбранное время только что занято другим клиентом».

### ТС-04: Валидация даты бронирования
* **Предусловия:** Форма создания записи открыта.
* **Шаги:**
  1. Указать в поле даты вчерашний день.
  2. Попробовать отправить форму.
* **Ожидаемый результат:** Форма блокируется атрибутом `min` на клиенте, либо сервер отклоняет запрос с ошибкой валидации. Запись в базу не вносится.

### ТС-05: Изменение статуса записи администратором
* **Предусловия:** В журнале есть запись со статусом «Ожидает».
* **Шаги:**
  1. В строке записи нажать кнопку «Подтвердить».
* **Ожидаемый результат:** Статус записи меняется на «Подтверждена», цвет индикатора меняется на зеленый.

### ТС-06: Отмена бронирования с освобождением времени
* **Предусловия:** Существует подтвержденная запись на конкретное время.
* **Шаги:**
  1. В строке записи нажать «Отменить».
  2. Подтвердить действие в модальном окне браузера.
  3. Перейти в создание записи на этот же день.
* **Ожидаемый результат:** Статус записи в журнале становится «Отменена» (красный маркер). Отмененный временной слот снова становится доступным для выбора другими клиентами.

### ТС-07: Перенос времени записи (Reschedule)
* **Предусловия:** В журнале есть активная запись.
* **Шаги:**
  1. Нажать кнопку «Перенести» в строке записи.
  2. На открывшейся странице указать новую дату и выбрать другое время.
  3. Нажать «Сохранить изменения».
* **Ожидаемый результат:** Запись обновляется, старое время освобождается, новое бронируется. Изменение фиксируется в логах.

### ТС-08: Фильтрация журнала записей
* **Предусловия:** В журнале присутствуют записи с разными датами и статусами.
* **Шаги:**
  1. В блоке фильтров выбрать конкретный статус (например, «Отменена»).
  2. Нажать кнопку «Применить фильтры».
* **Ожидаемый результат:** В таблице скрываются все строки, кроме записей со статусом «Отменена».

### ТС-09: Формирование аналитических отчетов
* **Предусловия:** В базе данных есть несколько созданных и отмененных записей.
* **Шаги:**
  1. Перейти во вкладку «Отчёты» главного меню.
* **Ожидаемый результат:** Страница открывается без ошибок, отображаются две таблицы: статистика записей по дням и рейтинг популярности курсов с корректными числовыми значениями.

### ТС-10: Экспорт данных в формат CSV
* **Предусловия:** Страница отчетов открыта, таблицы не пусты.
* **Шаги:**
  1. Нажать на кнопку «Скачать CSV-отчет».
* **Ожидаемый результат:** Браузер инициирует скачивание файла `report.csv`. При открытии файла данные структурированы и соответствуют таблицам на сайте.
